React-native从入门到放弃

发表于 2018-12-15 | 分类于 React native |
字数统计: 740字 | 阅读时长 ≈ 3分

React native 实在太多坑，
项目跑起来就废了很大劲
全部按照中文网的步骤配置 mac 的 Android 和 IOS

开始记录一下遇到的问题吧

转入 IOS
0.50 以上版本按照官网配置无法运行，
退回 0.44.3 版本

需要关闭安卓模拟器，或者重启几次，再执行

1、Error watching file for changes: EMFILE 及`brew link autoconf automake

解决方法

  • 卸载原来安装的 watchman:brew uninstall –force watchman

  • 删除原来的安装文件:rm -rf /usr/local/var/run/watchman/

  • 重新安装 watchman:brew install watchman

2、报错 No bundle URL present，报错信息要看运行时弹出的窗口

解决方法：删除 ios/build 文件夹，重新运行

3、Android，不启动 android studio,直接运行模拟器

执行 emulator -avd First(创建的模拟器的名字),
报错：PANIC: Missing emulator engine program for ‘x86’ CPU.

解决方法：直接打开一个终端，
cd ~进入电脑根路径
open Library 打开文件夹 Library
找到 Android/sdk，将 emulator 文件夹的东西复制粘贴到 tools。
注意

  • 试了一下，貌似每次重新执行都要这么操作一下，嫌麻烦的可以通过 android studio 打开模拟器
  • 如果想要恢复，可以先保存一份 tools 里面的文件，压缩一下就可以

4、模拟器内调用开发者菜单

IOS：command+D
Android：command+M

Enable Live Debugging 自动刷新

5、IOS 模拟器选中输入框不弹出键盘

 解决方法： 1.选中模拟器，屏幕上的菜单 Hardware-》keyboard-》Connect Hardware keyBoard 取消选中

但是取消选中之后，电脑键盘快捷键调用开发者模式失效
2.第二中，不取消选中第二个选项Connect Hardware keyBoard，每次需要调用菜单的时候选上图第三个操作，或快捷键 commond+K

6、IOS 配置 APP 的图标、启动页面 、修改名字

  • 使用 Xcode 打开 ios 工程

  • 将图片和启动页面放置到工程目录下 ios/新建的项目名称/Images.xcassets

  • 设置

  • 修改 App 名字
    ios/新建的项目名称/info.plist

7、Android 配置 APP 的图标、修改名字

  • 使用 Android studio 打开 android 工程
  • android/app/src/res/values/strings.xml
  • 将图标文件夹放入工程目录
    drawable 放入 android/app/src/res
    在 app/src/main/AndroidManifest.xml 中修改 android:icon 的值，替换为放入的 icon

8、添加 Mobx 状态管理

  • 下载 mobx 插件
  • 配置装饰器插件
  • .babelrc 增加配置

  • android 报错 can’t find varivable:Symbol
    原因是因为 mobx 的版本用的最新版本..用到了 Symbol 部分 es6 的 api 特性.
    把 mobx 降版本到 4.3.1 . mobx-React 降版本到 5.1.0 即可.

  • android 键盘把底部导航栏顶起
    解决：android/app/src/AndroidManifest.xml
    改变配置：android:windowSoftInputMode=”stateAlwaysHidden|adjustPan”>
