React-native之ScrollView

发表于 2018-12-26 | 分类于 React native |
字数统计: 2.7k字 | 阅读时长 ≈ 11分

ScrollView 是一个通用的可滚动的容器，可以包含多个组件和视图
不仅可以垂直滚动也可以水平滚动

ScrollView

效果

注意事项

  • 必须有一个固定的高度
  • 不建议直接  设置高度
  • 使用 flex:1，使其自动填充父容器的剩余空间。前提条件是所有的父容器本身也设置了 flex 或指定高度。
  • ScrollView 内部的其他响应者尚无法阻止 ScrollView 本身成为响应者。
  • 只适用于简单数据的渲染，逻辑复杂的数据可以使用 Flatlist

常用属性

  • horizontal
    默认是垂直方向上排成一列
    horizontal={true}，子视图会在水平方向上排列
    效果

  • alwaysBounceVertical
    IOS 属性
    默认为 true，垂直方向即使内容比滚动视图本身还要小，也可以弹性拉动。
    当 horizontal={true}时默认值为 false,否则为 true

  • alwaysBounceHorizontal
    IOS
    默认为 true，即使水平方向比滚动视图小，也可以弹性拉动。
    当 horizontal={true}时默认值为 false,否则为 true

  • keyboardDismissMode
    用户拖动视图的时候，是否要隐藏软键盘
    通用值：

    • none（默认值），拖动时不隐藏软键盘
    • on-frag，开始拖拽的时候隐藏软键盘
       IOS：
    • interactive, 软键盘在拖动的时候逐渐消失，并且上滑会恢复键盘。android 设置此属性和 none 效果一致

  • keyboardShouldPersistTaps
    点击 scrollView 后是否收起软键盘

    • never(默认值)，点击 TextInpt 以为的子组件会使当前的软键盘收起。子元素不会收到点击事件
    • always，键盘不会收起，ScrollView 也不捕捉到点击事件，但子组件可以捕获
    • handled，当点击事件被子组件捕获时，键盘不会自动收起。切换 TextInput 时键盘可以保持状态，多个 TextInput 的 scrollView 可以使用这个组件。

  • pagingEnabled
    当值为 true 时，滚动条会停在滚动视图尺寸的整数倍位置。
    默认为 false，可以用在水平分页上。
    垂直分页在 Android 上不支持

  • refreshControl
    指定 RefreshControl 组件，用于为 ScrollView 提供下拉刷新功能，只能用于垂直视图，即 horizontal 不能为 true

  • removeClippedSubviews(使用特性)
    默认为 true，屏幕之外的子视图（overflo 为 hidden）被移除。可以提高滚动性能

  • scrollEnable
    默认为 true，为 false 的时候禁止用户滚动，但是可以调用 scrollTo 滚动

  • showHorizontalScrollIndicator
    默认为 true，当 horizontal={true}显示水平方向的滚动条
    为 false 时不显示水平滚动条

  • showVerticalScrollIndicator
    默认为 true，垂直滚动的时候显示垂直方向的滚动条
    为 false 时不显示垂直滚动条

  • enFillColor
    滚动视图占据比实际内容更多的空间，可以使用此属性指定某种颜色填充多余的空间。

  • automaticallAdjustContentInsets
    当滚动视图放在一个导航条或工具条后的时候，IOS 系统是否自动调整内容的范围。
    默认为 true。
    如果你的 ScrollView 或 FlatList 的头部出现莫名其妙的空白，尝试此属性设为 false

  • overScrollMode(Android 没发现效果)
    覆盖默认的 overScroll 模式

    • auto 默认值，允许用户在内容超出视频高度后可以滚动
    • always，无论内容尺寸，用户始终可滚动视图
    • never，始终不允许用户滚动视图

  • bounces （IOS）
    为 true（默认）时，范围比滚动视图本身大，到达内容尾部的时候，可以弹性拉动一截。
    为 false，尾部的所有弹性都会被禁用。即使 alwaysBounce 属性为 true

  • bouncesZoom（IOS）
    为 true 时，使用手势缩放内容可以超过 min/max 限制，然后在手指抬起之后弹回 min/max 的缩放比例。
    为 false 时，缩放不能超出限制

  • canCancelContentTouches（IOS）
    为 false 时，一旦子节点响应触摸操作，即使手指开始移动也不会拖动滚动视图。
    默认为 true

  • centerContent（IOS）
    值为 true 时，如果滚动视图的内容比视图本身小，则会自动把内容居中放置。当内容比滚动视图大的时候，此属性没有作用。
    默认为 false

  • contentInset（IOS）
    内容范围相对视图边缘的坐标。默认为{top:0,left:0,bottom:0,right:0}

  • contentOffset（IOS）
    用来手动设置初始的滚动坐标，默认值{x:0,y:0}

  • decelerationRate
    一个浮点数，决定用户抬起手指后，滚动视图减速停下的速度。
    可以设为 normal 或 fast。分别对应是 IOS 上的 UIScrollViewDecelerationRateNormal 和 UIScrollViewDecelerationRateFast

    • normal：IOS 上是 0，998，Android 上是 0.985（默认值）
    • fast：0.99

  • directionalLockEnabled IOS）
    为 true 时，滚动视图在拖拽的时候会锁定只有垂直或水平方向可以滚动，
    默认为 false

  • indicatorStyle（IOS）
    设置滚动条的样式，

    • default 默认值，等同 black
    • black 黑白滚动条
    • white 白色滚动条

  • maximumZoomScale（IOS）
    允许的最大缩放比例。默认为 1.0

  • minimumZoomScale (IOS)
    允许最小的缩放比例。默认为 1.0

  • pinchGestureEnabled(IOS)
    设置为 true 时，ScrollView 会允许用过使用双指缩放操作。默认为 true

  • scrollEventThrottle（IOS）
    控制滚动过程中，scroll 事件被调用的频率（单位是每秒的实际数量）。
    数值越小越能及时追踪滚动位置，不过会带来性能问题。
    由于 JS 事件循环需要和屏幕刷新率同步，设置 1-16 之间的数值不会有实质区别。
    默认为 0，每次视图滚动，scroll 事件只会被调用一次。

  • scrollIndicatorInsets（IOS）
    决定滚动条距离视图边缘的坐标。这个值应该和 contentInset 一样
    默认为{top:0,left:0,right:0,bottom:0}

  • scrollToTop
    为 true 时，点击状态栏的时候视图会滚动到顶部。
    默认为 true

  • snapToInterval
    数字类型
    设置了此属性，会让滚动视图停止后，停止在 snapToInterval 的倍数的位置。
    可以在一些子视图比滚动视图小的时候用于实现分页，需要与 snapToAlignment 组合使用。

  • snapTopAlignment
    当设置了，snapToInterval，snapToAlignment 会定义停住点与滚动视图之间的关系

    • start（默认）会将停住点对齐在左侧（水平）或顶部（垂直）
    • center 会将停住点对齐到中间
    • end 会将停驻点对齐到右侧（水平）或底部（垂直）

  • zoomScale
    滚动视图内容当前的缩放比例。默认为 1.0

  • onContentSizeChange
    ScrollView 内部可滚动内容的视图发生变化的时候调用。
    参数为内容视图的宽高:(contentWidth,contentHeight)
    此方法是通绑定在内容容器上的 onLayout 来实现

  • onMomentumScrollBegin
    滚动动画开始的时候调用

  • onMomentumScrollEnd
    滚动动画结束时调用

  • onScroll
    在滚动过程中，每帧最多调用一次此回调函数，调用的频率可以使用 scrollEventThrottle 属性来控制

  • onScrollBeginDrag
    用户开始拖动此视图调用此函数

  • onScrollBeginEnd
    用户停止拖动视图的时候调用此函数

常用方法

  • scrollTo()
    (y: number | { x?: number, y?: number, animated?: boolean }, x: number, animated: boolean)
    滚动到指定的 x,y 偏移处。第三参数为是否开启平滑滚动视图
    scrollTo({x:0,y:0,animated:true})

  • scrollToEnd(([options]:object))
    滚动到视图底部（水平方向的视图则滚动到最右边）
    加上动画参数 scrollToEnd({animated:true}),启动平滑滚动动画。
    scrollToEnd({animated:false})立即跳转。
    不使用参数，默认启动动画
